CAUSA RAÍZ:
-----------
Las Edge Functions NO usan supabase.auth.getUser(jwt), sino que:
1. Hacen fetch manual a /auth/v1/user
2. Envían Authorization header con "Bearer <jwt>"
3. Envían apikey header con valor de SUPABASE_ANON_KEY
4. Supabase Auth compara la signature del JWT con el apikey
5. Si no coinciden → 401 Unauthorized
6. Si SUPABASE_ANON_KEY no está configurado → apikey = "" → 401

POR QUÉ OCURRE AuthSessionMissingError:
---------------------------------------
1. El SDK de Supabase en el frontend intenta refrescar el token
2. La Edge Function rechaza el JWT con 401
3. El frontend interpreta esto como "sesión perdida"
4. Dispara AuthSessionMissingError
5. El usuario es deslogueado o redirigido

----------
- Logs del frontend muestran: "JWT validation failed"
- Logs del backend muestran: 401 Unauthorized
- Error consistente en todas las llamadas
- Patrón de validación manual propenso a errores
